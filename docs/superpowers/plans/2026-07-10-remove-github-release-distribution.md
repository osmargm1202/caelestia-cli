# Remove GitHub Release Distribution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove GitHub Release binary distribution and restore the source-built Rust package as the default so Cachix handles binary reuse.

**Architecture:** `caelestia-cli` remains a normal `buildRustPackage` derivation. `caelestia-shell` continues consuming `packages.${system}.default`; Cachix substitutes that exact derivation when available. Release workflow, fetchurl package, and obsolete distribution documentation disappear.

**Tech Stack:** Nix flakes, Rust/Cargo, GitHub Actions removal, Cachix substitution.

## Global Constraints

- Do not modify Rust command behavior.
- Do not modify or delete the historical `v0.4.2` GitHub release/tag.
- Do not rewrite Git history.
- Do not include unrelated dirty files from `/home/osmarg/Hobby/nixos`.
- Keep `caelestia-cli` source build available as `packages.${system}.caelestia-cli` and make it `packages.${system}.default`.
